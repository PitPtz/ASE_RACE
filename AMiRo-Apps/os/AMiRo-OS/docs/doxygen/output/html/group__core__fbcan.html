typedef void(* aos_fbcan_enqueuecb_t) (aos_fbcan_txframe_t *frame, aos_fbcan_txframe_t *queue_front, aos_fbcan_txframe_t *queue_back, void *params)

Enqueue callback function type.

Parameters
[in]frameFrame to enqueue. Must not be NULL.
[in,out]queue_frontPointer to the first element in the Queue. Must not be NULL. If the frame is prepended, the prev pointer must be modified accordingly.
[in,out]queue_backPointer to the last element in the queue. Must not be NULL. If the frame is appended, the next pointer must be modified accordingly.
[in]paramsOptional parameters.

◆ aosFBCanSleep()

void aosFBCanSleep ( AosFBCANDriver *  fbcanp)

Enters the sleep mode.

This function puts the CAN driver in sleep mode and broadcasts the sleep_event event source.

Precondition
In order to use this function the option CAN_USE_SLEEP_MODE must be enabled and the CAN_SUPPORTS_SLEEP mode must be supported by the low level driver.
Parameters
[in]fbcanpPointer to the aosBufferedCanDriver object.

◆ aosFBCanTransmit()

aos_status_t aosFBCanTransmit ( AosFBCANDriver *  fbcanp,
const CANTxFrame *  frame 
)

CAN frame transmission.

The specified frame is queued for transmission. If the hardware queue is full, the frame is enqueued in the transmit queue. The frame is always enqueued in either queue, because it had to be acquired from the transmit buffer before.

Parameters
[in]fbcanpPointer to the AosFBCANDriver object.
[in]framePointer to the CAN frame to be transmitted. Must have been acquired using aosFBCanAcquireTxFrame().
Returns
The operation result.
Return values
AOS_SUCCESSThe frame has been queued to the hardware.
AOS_WARNINGThe frame has been queued to the transmit buffer.
