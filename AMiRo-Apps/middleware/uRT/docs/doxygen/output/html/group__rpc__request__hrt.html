Detailed Description

Request with hard real-time constraints.

This is the least efficient request type, which behaves similarly to FRT requests. If timing constraints are violated, however, this is considered a severe system failure and a µRT panic is provoked immediately.

inline

Retrieve an HRT request.

Parameters
[in]requestThe request to retrieve. Must not be NULL.
[in]policyPolicy how to retrieve the request.
[out]bytesSize of the payload to be returned by reference (optional). May be NULL.
[out]latencyThe latency to be returned by reference (optional). May be NULL.
Returns
Status code indicating the result of the function call.
Return values
URT_STATUS_OKRequest was retrieved normally. The request is locked.
URT_STATUS_REQUEST_BADOWNERRequest had to be removed from a service. If policy was set to URT_REQUEST_RETRIEVE_ENFORCING, the request is locked.
URT_STATUS_REQUEST_LOCKEDRequest could not be acquired due to a locked mutex. Only occurs if the policy was set to URT_REQUEST_RETRIEVE_LAZY. Request is not locked.
URT_STATUS_DEADLINEVIOLATIONRequest was retrieved, but deadline has been violated. The request is locked. Only occurs for FRT and HRT requests.
URT_STATUS_JITTERVIOLATIONRequest was retrieved, but jitter has been jiolated. The request is locked. Only occurs for FRT and HRT requests.
