urt_hrtrequest_recovery_f urt_hrtrequest::callback

Callback to be executed in case a timing violation has occurred.

If set to NULL, urtCorePanic() will be executed as default. The callback will only be executed if the current value of the violationCode is URT_STATUS_OK.

◆ violationCode

urt_status_t urt_hrtrequest::violationCode

Status code indicating the reason for the first timing violation.

Its value is initialized with URT_STATUS_OK. Whenever a recovery attempt is initiated, this value is set to the status code, indicating the cause of the violation (i.e. deadline, jitter or rate). As long as its value is not reset to URT_STATUS_OK, no further attempts will be performed. It is thus th resposnibility of the recovery logic to reset this variable on success.
