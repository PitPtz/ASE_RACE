Detailed Description

Subscribers with hard real-time constraints.

This is the least efficient subscriber type, which behaves similarly to FRT subscribers. If timing constraints are violated, however, this is considered a severe system failure and a µRT panic is provoked immediately.

Subscribe to a topic.

Parameters
[in]subscriberThe HRT subscriber which shall subscribe to a topic. Must not be NULL. Must not be associated to a topic.
[in]topicThe topic to subscribe to. Must not be NULL.
[in]maskMask of event where the subscriber registers to. Must not be 0.
[in]deadlineMaximum latency to consume messages. A value of 0 indicates that latency is of no concern.
[in]jitterMaximum allowed jitter (difference between maximum and minimum latency) when consuming messages. A value of 0 indicates that jitter is of no concern.
[in]rateExpected minimum rate of new messages (= maximum time between consecutive messages). A value of 0 indicates, that rate is of no concern.
[in]recoverycbCallback to be executed when a timing constraint has been violated. If NULL, urtCorePanic() will be called instead.
[in]paramsOptional parameters to be passed to the recovery callback.
Returns
Always returns URT_STATUS_OK.
