Detailed Description

Publishers emit messages.

Publishers emit messages via an associated topic, thereby setting all meta information (e.g. time) and payload.

Enumeration Type Documentation

◆ urt_publisher_publishpolicy

enum urt_publisher_publishpolicy

Policies for publishing messages.

Enumerator
URT_PUBLISHER_PUBLISH_LAZY 

Abort if a locked mutex or a blocking HRT subscriber is detected.

URT_PUBLISHER_PUBLISH_DETERMINED 

Abort only if a blocking HRT subscriber is detected.

URT_PUBLISHER_PUBLISH_ENFORCING 

Always publish, even if it takes a while.

Publish a message.

Parameters
[in]publisherPointer to the publisher to use. Must not be NULL.
[in]payloadPointer to the data to be published. May be NULL for messages without payload.
[in]bytesSize of the payload in bytes.
[in]tTimestamp to be set for the message information. Must not be NULL.
[in]policyPolicy how to publish the message.
[in]timeoutTimeout delay in case the message cannot be published. Not available if URT_PUBSUB_PUBLISHER_PUBLISH_TIMEOUT is false.
Returns
Status code indicating the result of the function call.
Return values
URT_STATUS_OKMessage was published successfully.
URT_STATUS_PUBLISH_LOCKEDAssociated topic is currently locked. Only occurs if the policy was set to URT_PUBLISHER_PUBLISH_LAZY.
URT_STATUS_PUBLISH_BLOCKEDA pending HRT subscriber prohibits reutilization of the next message. Only occurs if the policy was not set to URT_PUBLISHER_PUBLISH_ENFORCING.
URT_STATUS_PUBLISH_TIMEOUTTimeout occurred. Can only occur if URT_PUBSUB_PUBLISHER_PUBLISH_TIMEOUT is true.
URT_STATUS_PUBLISH_OBSOLETEThe information carried by the message is too old. Can only occur if URT_CFG_PUBSUB_TOPIC_BUFFERORDER is URT_PUBSUB_TOPIC_BUFFERORDER_MSGTIME.
