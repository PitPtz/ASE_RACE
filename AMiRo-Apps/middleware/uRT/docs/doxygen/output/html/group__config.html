Detailed Description

µRT is configured via several feature flags and settings.

Flags can be used to deactivate entire subsystems (e.g. publish- subscribe or RPC), but also to configure those subsystems in detail. Feature flags and settings are evaluated at build time to allow for high level of optimization by compiler and linker.

#define URT_CFG_OSAL_CONDVAR_TIMEOUT   false

Flag to enable timeout functionality for condition variables.

If the OS supports this feature, urtPublisherPublish() can be called with a timeout. Otherwise it will block indefinitely under certain conditions.

◆ URT_CFG_PUBSUB_TOPIC_BUFFERORDER

#define URT_CFG_PUBSUB_TOPIC_BUFFERORDER   URT_PUBSUB_TOPIC_BUFFERORDER_MSGTIME

Method how messages are organized at topics.

Possible values are:

  • URT_PUBSUB_TOPIC_BUFFERORDER_MSGTIME Messages are ordered by their timestamps. Potentially linear computational complexity, but correct order.
  • URT_PUBSUB_TOPIC_BUFFERORDER_PUBTIME Messages are ordered by the times they are published. Constant computational complexity, but potentially "wrong" order.
